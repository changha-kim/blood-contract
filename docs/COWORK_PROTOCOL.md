# COWORK_PROTOCOL
> 목적: Cowork 호출이 헷갈리지 않게 ‘입력/출력/빈도’ 고정.

## Hard rules
- Cowork 입력 컨텍스트는 **항상** `docs/CONTEXT_BRIEF.md` + `docs/TEAM_B_REQUESTS_TODAY.md` 2개만.
- 과거 대화/로그/코드 원문 붙이지 않기 (필요 시 BRIDGE가 10줄 요약으로 제공)
- Cowork는 **생성/창작**만 (중개/정리/PR/요약은 BRIDGE가 담당)

## Call schedule
- 기본: 하루 1회 (필요 시 2회)
- 추가 질문은 ‘하루 1번’으로 몰아서

## Output format
- Cowork 산출물은 반드시 `docs/TEAM_B_DELIVERABLES_TODAY.md`에 아래 섹션으로만 작성:
  - Deliverables (bullets)
  - Decisions needed (max 3)
  - File list (paths)
  - Next actions (max 5)
