# BC_NARRATIVE_KO_UI_MICROCOPY_v002
> Assist Dept — Narrative Team | 2026-02-19 | v002

## 목적
Blood Contract PoC의 **핵심 전투 상태 UI 텍스트**를 한국어로 확정하기 위한 후보 2안 제시.
LOCK 큐, Telegraph 예고, Execute 집행 등 플레이어가 HUD에서 보게 되는 단어를 정의하고, **HUD 표시용** vs **로그/개발용** 구분 규칙을 명시한다.

> 관련 Open Question (ASSIST_MEMORY): "LOCK을 HUD에 한국어로 표기할지, 영어 LOCK를 유지할지 결정 필요"

---

## 사용 규칙: HUD 표시 vs 로그/개발용

| 구분 | 원칙 | 예시 |
|---|---|---|
| **HUD 표시** | 짧고 감각적인 한국어. 게임 세계관과 일치. 1–3 글자 선호. | "확정", "예고" |
| **로그/개발용** | 영어 snake_case 기술 식별자. 코드와 직결. 번역 불필요. | `commit_enter`, `telegraph_start` |
| **혼용 금지** | 같은 팝업/레이블에 한국어 HUD 단어 + 영어 코드 혼재 금지. | ❌ "LOCK(확정)" |
| **툴팁/튜토리얼** | 기술 설명 목적이면 영어 용어 병기 허용 (첫 등장 1회만). | "확정 (LOCK): 적의 방향이 고정됩니다." |

## 핵심 상태 단어 후보

### 1. COMMIT / LOCK (적의 의도가 확정된 상태)

#### 안 A: **"확정"**
- 뜻: 결정되어 바꿀 수 없음.
- 장점: 직관적. "적의 행동이 확정됐다"는 의미가 명확. 한국 게임 사용자에게 친숙.
- 단점: 게임적 긴박감이 약함. "행정 용어" 느낌.
- HUD 예시: 적 머리 위 자물쇠 아이콘 + "확정" 라벨 (선택적 표시)
- 개발 로그: `commit_enter` (변경 없음)

#### 안 B: **"봉인"**
- 뜻: 봉해서 열 수 없음. 혈계/계약 세계관과 직결.
- 장점: Blood Contract 세계관(계약·봉인)에 어울림. 시각적으로 강렬.
- 단점: "봉인 = 막힘"으로 해석될 수도 있음 (적이 묶인 것인지 행동이 정해진 것인지 혼동).
- HUD 예시: 자물쇠 아이콘 + "봉인" 라벨
- 개발 로그: `commit_enter` (변경 없음)

> **추천:** 안 A "확정" (TC01 통과 목적상 직관성 우선. 세계관은 아이콘으로 보완.)
> 최종 결정은 Main team(OpenClaw)이 내린다.

---

### 2. TELEGRAPH (예고/예비동작)

#### 안 A: **"예고"**
- 뜻: 미리 알림.
- 장점: 정확한 기능 묘사. 짧고 명확.
- 단점: 영화·공지 뉘앙스가 있음.
- HUD 예시: (HUD에 표시하지 않는 것이 기본 — 아이콘만으로 충분할 수 있음)
- 개발 로그: `telegraph_start`

#### 안 B: **"조짐"**
- 뜻: 일이 일어날 기미/낌새.
- 장점: 불길한 분위기. Blood Contract 세계관 적합.
- 단점: "조짐이 보인다"는 묘사적 단어라 UI 라벨로 짧게 쓰기엔 어색할 수 있음.
- HUD 예시: (역시 아이콘 우선 권장 — 필요 시 튜토리얼에서만 사용)
- 개발 로그: `telegraph_start`

> **추천:** Telegraph는 HUD에 텍스트 미표시(아이콘 전용)를 우선 검토. 표시 필요 시 안 A "예고".

---

### 3. EXECUTE (적이 확정된 행동을 실행하는 순간)

#### 안 A: **"집행"**
- 뜻: 결정된 것을 행함. 계약 이행.
- 장점: Blood Contract 세계관(계약 집행) 완벽 부합. 무게감 있음.
- 단점: "집행" 단독으로 HUD에 뜨면 다소 낯설 수 있음.
- HUD 예시: Execute 페이즈에서 짧은 플래시 텍스트 "집행" (0.4s, 충분히 큰 폰트)
- 개발 로그: `execute_start`

#### 안 B: **"발동"**
- 뜻: 작동하기 시작함.
- 장점: 게임 장르 친숙어. 카드게임/MOBA 사용자에게 익숙.
- 단점: 세계관 색이 약함. 일반적.
- HUD 예시: "발동" 플래시 텍스트
- 개발 로그: `execute_start`

> **추천:** 안 A "집행" (세계관 일관성. 단, 플레이테스트에서 "뭔 뜻인지 모르겠다" 피드백 시 안 B로 교체.)

### 4. 기타 HUD 핵심 단어 후보

| 상황 | 안 A | 안 B | 개발 로그 키 | 비고 |

## SpikeWall 피해 확인 텍스트 (TC04 전용)

TC04 패스 기준: 플레이어가 "적이 데미지를 받았다"는 것을 즉시 인지해야 함.
아래 텍스트가 데미지 숫자 위에 짧게 표시됨 (0.5s, 큰 폰트, 빨간색).

- **추천 표시:** 데미지 숫자 단독 (예: "47") — 숫자가 충분히 크면 텍스트 불필요.
- **보조 표시 (옵션):** "관통!" 또는 "박힘!" (세계관 우선이면 "관통", 직관성 우선이면 "박힘")
- **금지:** "SPIKED", "DAMAGE" 등 영어 혼용 — KO-first 규칙 위반.
