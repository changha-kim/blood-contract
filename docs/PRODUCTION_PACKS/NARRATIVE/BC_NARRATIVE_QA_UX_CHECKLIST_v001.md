# BC_NARRATIVE_QA_UX_CHECKLIST_v001
> 목표: Narrative 텍스트가 실제 런/모바일 화면에서 깨지지 않는지 빠르게 검증.

## Must-pass (P0)
- [ ] HUD 고정 용어 6종이 전 구간에서 동일 표기(확정/집행/공명/파기/계약 실패/계약 이행)
- [ ] 실패/클리어 헤더가 화면 폭에서 잘리지 않음(줄바꿈/말줄임 확인)
- [ ] 토스트가 동시에 2개 이상 뜰 때 우선순위/겹침이 사용 가능 수준
- [ ] 모달/확인 문구가 1줄 규칙을 지킴(장문 금지)

## Nice-to-have (P1)
- [ ] "다시 계약"(재시작) 버튼이 반복 플레이에서 피로하지 않음
- [ ] 카드 선택 화면: "대가"/"이득" 라벨이 용어 충돌 없이 읽힘
- [ ] Tier3/4 토스트(공명/파기)가 ‘기분 좋은 변화’로 인지됨(플레이어 1문장 피드백 기록)

## Quick test script (10분)
1) 런 시작 → 첫 룸 진입 토스트 확인
2) 카드 선택 진입/리롤/확정 플로우 1회
3) Tier3/4 도달(가능하면) 시 토스트 확인
4) 실패 화면 진입(계약 실패) + 재시작
5) 클리어(계약 이행) 또는 Extract(중도 이행) 텍스트 확인
